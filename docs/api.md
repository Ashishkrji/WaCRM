# API Reference Manual — WaCRM Enterprise

This document lists the serverless REST API endpoints for the AI engine, knowledge-base indexing, and public signature portals.

---

## 🤖 1. AI Assistant Endpoints

### A. Test Chatbot Response
Allows sandbox testing of chatbot responses using your system instructions and database context.
*   **Endpoint**: `POST /api/ai/chat`
*   **Authentication**: Session cookie (Authenticated users only)
*   **Payload**:
    ```json
    {
      "message": "I want to request a quotation for a Shopify store",
      "conversationId": "38a329d4-c9f2-4e42-8869-7925c48b17a1",
      "contactId": "b1fa6623-1d90-410a-b302-3c16ee0a1740"
    }
    ```
*   **Response (200 OK)**:
    ```json
    {
      "reply": "I can certainly generate a custom quote for a Shopify store! Our standard packages range from ₹25,000 to ₹60,000. Would you like me to compile a quotation now?",
      "confidence": 0.94,
      "model": "nvidia/llama-3.1-nemotron-70b-instruct"
    }
    ```

### B. Streaming Completion
Streams tokens as they are generated by the model.
*   **Endpoint**: `POST /api/ai/stream`
*   **Payload**: Same as `/api/ai/chat`
*   **Response (200 OK)**: `text/event-stream` returning data chunks:
    ```text
    data: "I"
    data: " can"
    data: " help"
    ...
    data: "[DONE]"
    ```

## 📚 2. Knowledge Base & Ingestion Endpoints

### A. Fetch Ingested Documents
Lists all files and URLs ingested into the vector database.
*   **Endpoint**: `GET /api/ai/knowledge`
*   **Response (200 OK)**:
    ```json
    {
      "documents": [
        {
          "id": "e83cf312-d819-482a-a53c-f4b23ce81043",
          "title": "MaaJanki Pricing Sheet",
          "doc_type": "pdf",
          "status": "ready",
          "chunk_count": 14,
          "created_at": "2026-06-21T10:00:00.000Z"
        }
      ]
    }
    ```

### B. Ingest URL (Crawl Scraper)
Submits a public website URL to scrape and vectorize.
*   **Endpoint**: `POST /api/ai/knowledge`
*   **Payload**:
    ```json
    {
      "title": "Pricing Page",
      "doc_type": "website",
      "source_url": "https://maajanki.com/pricing"
    }
    ```
*   **Response (200 OK)**:
    ```json
    {
      "success": true,
      "document": { "id": "uuid", "status": "pending" }
    }
    ```

### C. Ingest Local File
Uploads a physical file, uploads it to Supabase Storage under the user's bucket path, and queues it for vector embedding.
*   **Endpoint**: `POST /api/ai/knowledge/upload`
*   **Content-Type**: `multipart/form-data`
*   **Payload**: Form containing `file` parameter (accepts `.pdf`, `.docx`, `.txt`).
*   **Response (200 OK)**:
    ```json
    {
      "success": true,
      "fileName": "pricing.pdf",
      "documentId": "e83cf312-d819-482a-a53c-f4b23ce81043"
    }
    ```

### D. Delete Document
Deletes a document from storage and purges all of its associated vector chunks from the `knowledge_embeddings` table.
*   **Endpoint**: `DELETE /api/ai/knowledge?id=<document_id>`
*   **Response (200 OK)**:
    ```json
    {
      "success": true
    }
    ```

## 💼 3. Public Client Portals (Quotes & Proposals)

These are public-facing REST endpoints used by the signature portals. They do not require agent credentials, but verify transactions using their secure transaction UUIDs.

### A. Proposals Portal
*   **Fetch Proposal**: `GET /api/public/proposal?id=<proposal_id>`
    *   **Response (200 OK)**: Returns the proposal details, client details, service description, status, and custom branding accent color.
*   **Sign Proposal**: `POST /api/public/proposal`
    *   **Payload**:
        ```json
        {
          "id": "33b2a29d-47fa-4b8c-8f92-5ac510b64d02",
          "signature": "John Doe"
        }
        ```
    *   **Response (200 OK)**:
        ```json
        {
          "success": true,
          "status": "signed",
          "signed_at": "2026-06-21T10:45:00Z"
        }
        ```

### B. Invoices/Quotes Portal
*   **Fetch Quotation**: `GET /api/public/quote?id=<quote_id>`
    *   **Response (200 OK)**: Returns the invoice breakdown, GST computations (18% local standard), payment status, and business identity.
*   **Record Payment & Sign**: `POST /api/public/quote`
    *   **Payload**:
        ```json
        {
          "id": "c1f1092a-3c72-4e89-912f-9811c0c66b44",
          "signature": "John Doe",
          "paid": true
        }
        ```
    *   **Response (200 OK)**:
        ```json
        {
          "success": true,
          "status": "paid",
          "paid_at": "2026-06-21T10:45:00Z"
        }
        ```
