# Database Schema Book

WaCRM utilizes a dual-database architecture:
1. **Supabase (PostgreSQL)** for structured relational data, transactions, and RBAC.
2. **MongoDB Atlas** for unstructured data, high-volume logs, AI Memory, and Vector Search (RAG).

---

## 1. Supabase (PostgreSQL) Schema

### 1.1 `organizations`
Core tenant table for Multi-Organization support.
- `id` (uuid, pk)
- `name` (text)
- `billing_plan` (text) - e.g., 'free', 'pro', 'enterprise'
- `created_at` (timestamptz)

### 1.2 `users` (Managed by Supabase Auth)
- Maps directly to `employees` via `auth.users.id`.

### 1.3 `employees`
- `id` (uuid, pk, references `auth.users.id`)
- `organization_id` (uuid, fk)
- `role_id` (uuid, fk)
- `full_name` (text)
- `status` (text) - 'active', 'suspended'

### 1.4 `roles` & `permissions`
- **roles**: `id`, `name`, `organization_id`
- **permissions**: `id`, `resource`, `action`
- **role_permissions**: Many-to-many mapping.

### 1.5 `customers` (CRM Core)
- `id` (uuid, pk)
- `organization_id` (uuid, fk)
- `name` (text)
- `phone_number` (text) - unique per org
- `email` (text)
- `status` (text) - 'lead', 'active', 'churned'

### 1.6 `leads` & `deals` (Sales Pipeline)
- **leads**: `id`, `customer_id`, `source`, `score`, `status`
- **deals**: `id`, `customer_id`, `amount`, `stage` (e.g., 'prospecting', 'closed_won'), `expected_close_date`

### 1.7 Row Level Security (RLS) Strategy
Every table MUST have an `organization_id`.
RLS Policy standard:
```sql
CREATE POLICY "tenant_isolation" ON public.customers
FOR ALL USING (organization_id = current_setting('app.current_org_id')::uuid);
```

## 2. MongoDB Atlas Schema

### 2.1 `conversation_memory`
Stores complete WhatsApp chat histories for AI context window retrieval.
- `_id` (ObjectId)
- `organization_id` (String - UUID)
- `customer_id` (String - UUID)
- `messages` (Array)
  - `role` ('user', 'assistant', 'system')
  - `content` (String)
  - `timestamp` (Date)
- `summary` (String) - Periodically generated by AI to compress history.

### 2.2 `knowledge_base` (Vector Store)
Stores chunked documents for RAG.
- `_id` (ObjectId)
- `organization_id` (String - UUID)
- `document_title` (String)
- `chunk_text` (String)
- `embedding` (Array of Floats) - 1536 dims (OpenAI) or corresponding NVIDIA dims.
- *Index*: Atlas Vector Search Index on `embedding` field.

### 2.3 `ai_logs`
High-volume audit trail of every AI generation.
- `_id` (ObjectId)
- `prompt_tokens` (Number)
- `completion_tokens` (Number)
- `provider` (String) - 'nvidia', 'openai'
- `latency_ms` (Number)
- `timestamp` (Date)
